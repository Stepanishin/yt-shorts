# Автоматическая публикация по расписанию

## Как это работает

Сервер **автоматически** проверяет запланированные видео каждые 10 минут и публикует их на YouTube, когда приходит время. Никаких дополнительных настроек не требуется!

### Принцип работы:

1. **При старте сервера** запускается встроенный планировщик
2. **Каждые 10 минут** планировщик проверяет все запланированные видео всех пользователей
3. **Автоматически публикует** те видео, время которых уже пришло
4. **Обновляет статусы** в базе данных

## Использование:

### 1. Запланировать видео:

1. Создайте видео в Video Constructor
2. Нажмите кнопку **"🕐 Запланировать"** 
3. Выберите дату и время
4. Сохраните

### 2. Видео автоматически опубликуется:

- Сервер проверит в ближайшие 10 минут после указанного времени
- Опубликует на YouTube автоматически
- Обновит статус на "published"

### 3. Проверить статус:

Откройте `/dashboard/scheduled` - там увидите все запланированные видео и их статусы

## Важно для деплоя на Digital Ocean:

### Digital Ocean App Platform:

Убедитесь что используете **постоянно работающий сервис** (не serverless функции).

В `app.yaml` должен быть тип `web`, а не `functions`:

```yaml
services:
  - name: web
    instance_count: 1
    instance_size_slug: basic-xxs
    http_port: 3000
    run_command: npm start
```

### Digital Ocean Droplet (VPS):

Просто запустите сервер:

```bash
npm run build
npm start
```

Планировщик запустится автоматически.

### PM2 (рекомендуется для VPS):

```bash
npm install -g pm2
pm2 start npm --name "shorts-generator" -- start
pm2 save
pm2 startup
```

## Что делать если видео не публикуются:

### 1. Проверьте логи сервера:

```bash
# Для PM2
pm2 logs shorts-generator

# Для Digital Ocean App Platform
# Откройте Runtime Logs в дашборде
```

Вы должны видеть сообщения `📅 Checking for scheduled videos...` каждые 10 минут.

### 2. Убедитесь что сервер работает постоянно:

Планировщик работает только пока работает сервер. Если сервер перезапускается/выключается, планировщик останавливается.

### 3. Проверьте авторизацию YouTube:

Пользователь должен быть авторизован в YouTube через `/dashboard/settings`.

### 4. Проверьте базу данных:

Откройте `/dashboard/scheduled` - видео должно быть в статусе "planned" и время должно быть в прошлом.

## FAQ:

**Q: Можно изменить интервал проверки?**
A: Да, откройте `lib/scheduler.ts` и измените `CHECK_INTERVAL`:
```typescript
const CHECK_INTERVAL = 5 * 60 * 1000; // 5 минут
```

**Q: Что если сервер упал во время публикации?**
A: При следующем запуске сервера планировщик проверит все просроченные видео и опубликует их.

**Q: Будет ли публикация если на сайте никого нет?**
A: Да! Планировщик работает на сервере независимо от того, есть посетители или нет.

**Q: Сколько ресурсов это занимает?**
A: Минимально. Проверка базы данных каждые 10 минут - это очень легкая операция.
